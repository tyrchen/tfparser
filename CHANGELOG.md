### Phase 9 — Hardening (M6)

- **`DefaultPipeline`**: new top-level orchestrator wiring discovery →
  loader → projection → terragrunt → evaluator → graph → provider. The CLI
  is now a thin wrapper around `tfparser_core::DefaultPipeline`.
- **CLI parity with the library**: `tfparser parse` now emits all four
  Parquet tables (`resources`, `dependencies`, `components`, `modules`) and
  the manifest, plus support for `--environment`, `--region`,
  `--profile-map`, `--aws-config`, `--var K=V`, `--allow-env`,
  `--env-mode`, `--strict-providers`, `--compression`, `--zstd-level`,
  `--tables`.
- **`CompressionOpt::zstd(level)`** — fallible constructor that rejects
  out-of-range zstd levels at the boundary (closes deferred P-016).
- **`ValidationError::Range`** variant for numeric-range validation.
- **CLI command-line redaction** — `--*-token=`, `--*-secret=`,
  `--*-password=` flag values are redacted before landing in the manifest
  (closes deferred P-020).
- **CLI manifest read** is now capped at 4 MiB via `Read::take` (closes
  deferred P-097).
- **Criterion bench harness** — `make bench` (and the
  `bench-save-baseline` / `bench-vs-baseline` Makefile targets) gates a
  10 % regression budget per spec 71 § 7.
- **Exporter cleanup**: `secondary::sorted_edges` retired in favour of
  borrowing the workspace's already-sorted edge vector with a
  `debug_assert!` invariant check (closes P-094). Module-count loop
  uses idiomatic `*entry.or_insert(0) += 1` (closes P-095).
- **Lib top-level doc** updated to reflect every phase landed.
